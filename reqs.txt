1. Introducción
1.1 Propósito del Documento
Este documento define los requerimientos técnicos completos para el desarrollo del DMS (Dealer Management System) de Elemotor, correspondiente al Punto 1 del plan de trabajo del Ecosistema Digital. El DMS es el backend central del ecosistema: la API REST, la base de datos MySQL y el panel de administración desde donde el equipo de Elemotor gestiona toda la operación digital.
1.2 Alcance
El DMS comprende tres componentes: (1) API REST que expone todos los datos y operaciones del negocio a los módulos consumidores (Web Inmersiva, Chatbot WhatsApp, n8n), (2) Base de datos MySQL con el esquema relacional completo, y (3) Panel de Administración web (admin.elemotor.com.co) donde el equipo interno gestiona inventario, importaciones, leads/cotizaciones, talleres aliados, precios de calculadora y contenido.

11. Glosario
Término
Definición
DMS
Dealer Management System. Sistema central de gestión del concesionario.
API REST
Interfaz de programación basada en HTTP con verbos GET, POST, PUT, DELETE.
JWT
JSON Web Token. Estándar para autenticación stateless.
RBAC
Role-Based Access Control. Autorización basada en roles de usuario.
ORM
Object-Relational Mapping. Capa de abstracción entre código y base de datos.
DTO
Data Transfer Object. Clase que define la estructura de datos de entrada/salida.
Prisma
ORM type-safe para Node.js/TypeScript con soporte MySQL.
NestJS
Framework de Node.js enterprise-grade con arquitectura modular.
Swagger
Herramienta para documentar APIs REST automáticamente.
Webhook
Callback HTTP que notifica a un sistema externo cuando ocurre un evento.
OTP
One-Time Password. Código de verificación de un solo uso.
Haversine
Fórmula matemática para calcular distancia entre coordenadas GPS.
bcrypt
Algoritmo de hasheo para contraseñas con salt automático.
Passenger
Servidor de aplicaciones que integra Node.js con Apache en cPanel.
ISR
Incremental Static Regeneration (relevante para la web que consume la API).
CORS
Cross-Origin Resource Sharing. Control de qué dominios pueden acceder a la API.
UTM
Urchin Tracking Module. Parámetros de tracking de campañas (source, medium, campaign).
CDN
Content Delivery Network. Red de distribución de contenido.
Lead
Prospecto comercial que muestra interés en un producto.
Funnel
Embudo de conversión: pending > contacted > responded > negotiation > closed.
